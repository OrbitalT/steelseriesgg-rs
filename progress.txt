## Learnings
(Patterns discovered during implementation)

### Protocol Analysis Patterns
1. **Zone-based foundation exists** - Good starting point for per-key expansion
2. **65-byte HID report structure** - Consistent across SteelSeries devices
3. **Interface 1 for keyboards** - Standard pattern for SteelSeries keyboard control
4. **Command + subcommand pattern** - 0x21, 0xFF suggests expandable command structure

### Technical Debt Identified
1. **Hardcoded zone counts** - Need dynamic detection based on actual device capabilities
2. **Limited error feedback** - No way to detect if commands actually succeeded
3. **No device state queries** - Cannot read current RGB settings or device mode
4. **Missing per-key infrastructure** - Fundamental gap for precise control
