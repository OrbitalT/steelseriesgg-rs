# Fix release.yml ‚Äî Add Windows & Arch builds

## Problem
`release.yml` only builds for Linux. The backup (`release-bak.yml`) built for Windows and Arch too, but also managed its own changelog/release creation ‚Äî duplicating what `release-please.yml` now handles.

## Current architecture
- **release-please.yml** ‚Äî on push to `main`, maintains `CHANGELOG.md` + crate version via Conventional Commits. Creates a GitHub Release with changelog notes when the release PR is merged.
- **release.yml** ‚Äî triggered by `v*` tags, builds Linux binary, uploads to the GitHub Release.
- **build-windows.yml / build-arch.yml** ‚Äî standalone `workflow_dispatch` builds; upload artifacts but don't attach to releases.

## Plan

Edit `.github/workflows/release.yml` to add two new build jobs (Windows, Arch) alongside the existing Linux job. Each job uploads its artifact directly to the GitHub Release via `softprops/action-gh-release@v3`. No `prepare` job needed ‚Äî release-please already creates the release with changelog notes.

### Changes to `release.yml`

1. **Update header comment** ‚Äî remove the note about Windows/Arch being separate manual builds.

2. **Keep existing `build` job** (Linux) ‚Äî no changes except pin toolchain to `1.94.1` for consistency with CI/MSRV.

3. **Add `build-windows` job**:
   - `runs-on: windows-latest`
   - `dtolnay/rust-toolchain@stable` with `toolchain: 1.94.1`, target `x86_64-pc-windows-msvc`
   - `Swatinem/rust-cache@v2`
   - `cargo build --release --locked`
   - Package: copy `ssgg.exe` ‚Üí `ssgg-windows-x86_64.exe`, compute SHA-256
   - Upload to release via `softprops/action-gh-release@v3` (same `if: startsWith(github.ref, 'refs/tags/')` guard)

4. **Add `build-arch` job**:
   - `runs-on: ubuntu-latest`, `container: archlinux:base-devel`
   - Install deps (`pacman -Syu --noconfirm`, `pacman -S --needed --noconfirm git rust systemd`)
   - Create unprivileged build user
   - Extract version from tag (`${GITHUB_REF_NAME#v}`), build source tarball, run `makepkg`
   - Compute SHA-256
   - Upload to release via `softprops/action-gh-release@v3`

5. **No `prepare` or `release` orchestration job** ‚Äî each build job uploads independently. Release-please already creates the GitHub Release with changelog body.
